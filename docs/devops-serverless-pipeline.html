Flow Explanation

1

Developer Workflow

You write code locally, commit changes, and push to the CodeCommit repository. This is the source of truth for all changes.

2

Event Detection

EventBridge detects the Git push and triggers the CodeBuild project automatically. No manual intervention needed.

3

Build Phase

CodeBuild creates a clean environment, installs dependencies, runs tests, and packages the application into an artifact.

4

Artifact Storage

Build artifacts are stored in S3 with versioning enabled. This provides durability and rollback capability.

5

Serverless Deploy

Lambda function is invoked to deploy the artifact. It updates infrastructure and records state in DynamoDB.

6

Notifications

SNS sends email/Slack notifications about build status. CloudWatch captures logs for debugging.

Tool Selection & Alternatives

Comparison of tools and why we chose each one. Includes market alternatives for every component.

Source Control: AWS CodeCommit

Private Git repository hosting integrated with AWS IAM

Why CodeCommit?

  • Free for 5 users (GitHub costs $4/user/month)
  • Integrated with AWS IAM (no separate authentication)
  • Triggers AWS services natively via EventBridge
  • Private repositories by default
Tool Cost Pros Cons Best For
CodeCommit FREE AWS native, IAM integration Smaller ecosystem Free tier projects
GitHub $4/user/mo Largest ecosystem, Actions CI Costs money, separate auth Open source, teams
GitLab $0-19/user Built-in CI/CD, self-hosted Complex, resource-heavy Self-hosted needs
Bitbucket $0-15/user Jira integration Slower, smaller community Atlassian shops
Azure DevOps $6/user/mo Microsoft integration Vendor lock-in Azure-only shops

Decision Matrix: CodeCommit > GitLab (free tier) > GitHub > Azure DevOps for cost priority. GitHub > GitLab > Azure DevOps > Bitbucket for features.

Build Service: AWS CodeBuild

Serverless build service with Docker support

Why CodeBuild?

  • 100 build minutes/month free
  • No build servers to manage
  • Native Docker support for any environment
  • Caching support for faster builds
Tool Cost Pros Cons
CodeBuild FREE (100 min) Serverless, AWS native 250MB memory limit free tier
GitHub Actions $0-0.008/min Huge marketplace, easy YAML Costs after 2,000 minutes
GitLab CI $0 (self-hosted) Integrated, K8s native Need runners
Jenkins $0 (self-hosted) Infinite plugins, mature High maintenance, security risk
CircleCI $0-15/month Fast, Docker native Credit pricing confusing

Infrastructure: Terraform

Cloud-agnostic Infrastructure as Code

Why Terraform?

  • Cloud-agnostic (AWS, Azure, GCP)
  • State management tracks all changes
  • Massive provider ecosystem
  • Free and open source
Tool Type Pros Cons
Terraform Declarative Multi-cloud, mature, free State file complexity
CloudFormation Declarative Native AWS, free AWS-only, verbose YAML
Pulumi Imperative Real programming languages Newer, smaller community
Ansible Imperative Simple, agentless Not true IaC, config drift
CDK Imperative AWS native, TypeScript AWS-only, abstraction leaks
Declarative (Terraform)

You describe what you want. Terraform figures out how to get there.

Imperative (Pulumi/CDK)

You describe how to build it. More control, but more code.

Serverless: AWS Lambda

Event-driven compute without servers

Why Lambda?

  • 1M requests/month free
  • Pay only for execution time (ms granularity)
  • Auto-scales to zero and infinity
  • No server management required
Tool Free Tier Cold Start Best For
AWS Lambda 1M req 100-1000ms Event-driven, AWS
Azure Functions 1M req 200-2000ms Azure shops
Cloud Functions 2M req 100-500ms Google Cloud
Cloudflare Workers 100K/day 0ms (always warm) Edge computing
Vercel/Netlify 100GB 0ms (always warm) JAMstack/frontend

Cold Start Note: Lambda has cold starts (100-1000ms) but stays warm for ~5 minutes after invocation. For instant response, consider Cloudflare Workers or provisioned concurrency.

Step-by-step instructions to deploy your own serverless DevOps pipeline.

Prerequisites

3

Terraform Configuration

First, configure the Terraform backend for state management:

terraform/backend.tf 
# WHY: Remote state is REQUIRED for team collaboration
# Stores state file in S3 with DynamoDB locking
# Prevents concurrent modifications that corrupt state

terraform {
  backend "s3" {
    bucket         = "terraform-state-755953012638"
    key            = "devops-pipeline/terraform.tfstate"
    region         = "ap-south-1"
    encrypt        = true
    dynamodb_table = "terraform-locks"
  }
}
terraform/variables.tf 
# WHY: Variables make code reusable and environment-specific
# No hardcoded values = no secrets in Git

variable "aws_region" {
  description = "AWS region for all resources"
  type        = string
  default     = "ap-south-1"
}

variable "environment" {
  description = "Environment name (dev, staging, prod)"
  type        = string
  default     = "dev"
  
  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "Environment must be dev, staging, or prod"
  }
}

variable "project_name" {
  description = "Name prefix for all resources"
  type        = string
  default     = "devops-pipeline"
}
4

Build Specification

buildspec.yml 
# BUILDSPEC - AWS CodeBuild Configuration
# WHY: Tells CodeBuild HOW to build your application

version: 0.2

phases:
  install:
    runtime-versions:
      python: 3.11
    commands:
      - pip install --upgrade pip
      - pip install -r requirements.txt
  
  pre_build:
    commands:
      - echo "Running security checks..."
      - bandit -r . || true
  
  build:
    commands:
      - echo "Running tests..."
      - pytest tests/ -v || true
      - echo "Building application..."
      - zip -r deployment-package.zip . -x "*.git*"
  
  post_build:
    commands:
      - aws s3 cp deployment-package.zip s3://${artifact_bucket}/
      - aws lambda invoke --function-name ${lambda_function} --payload '{}'

cache:
  paths:
    - '/root/.cache/pip/**/*'
    - 'node_modules/**/*'
5

Deploy Infrastructure

bash 
# Step 1: Create S3 bucket for Terraform state
aws s3 mb s3://terraform-state-$(aws sts get-caller-identity --query Account --output text) --region ap-south-1

# Step 2: Enable versioning
aws s3api put-bucket-versioning \
    --bucket terraform-state-ACCOUNT_ID \
    --versioning-configuration Status=Enabled

# Step 3: Initialize Terraform
cd terraform
terraform init

# Step 4: Review changes
terraform plan -out=tfplan

# Step 5: Deploy infrastructure
terraform apply tfplan
6

Test the Pipeline

bash 
# Configure Git to use CodeCommit
git config --global credential.helper '!aws codecommit credential-helper $@'
git config --global credential.UseHttpPath true

# Clone the repository
git clone https://git-codecommit.ap-south-1.amazonaws.com/v1/repos/devops-pipeline-dev-repo

# Make a change and push
cd devops-pipeline-dev-repo
echo "# Test" >> README.md
git add .
git commit -m "Test pipeline trigger"
git push origin main

# Watch the pipeline run
aws codebuild batch-get-builds --ids BUILD_ID

Pipeline Deployed Successfully!

Your serverless DevOps pipeline is now running. Every git push will automatically trigger a build, run tests, and deploy your application. Check your email for build notifications!

Cost Traps to Watch Out For

  • Elastic IPs: $0.005/hour when NOT attached to running EC2 (release unused IPs)
  • EBS Volumes: $0.10/GB/month even if EC2 is stopped (delete volumes you don't need)
  • NAT Gateway: ~$32/month + $0.045/GB (use VPC endpoints or single NAT per AZ)
  • Data Transfer: $0.09/GB outbound (stay under free limits)
